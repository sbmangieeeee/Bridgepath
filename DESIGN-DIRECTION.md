# Design Direction

Bridgepath uses an original layered storybook treatment: warm, adventurous, tactile, and recognisably Trinidad and Tobago without preschool styling or tourism shorthand.

The v1 world is a bounded illustrated map with large semantic location buttons and a keyboard/list alternative. Core navigation and learning remain DOM-first; no essential content is canvas-only.

## System principles

- One dominant action per screen, short text, large controls, visible focus, and explicit status.
- Replayable narration with persistent text; separate future controls for narration, music, and effects.
- Purposeful motion only, with reduced-motion support and no motion-only meaning.
- Responsive layouts, 200% zoom support, non-colour indicators, captions/transcripts, and no strict timers.
- WCAG 2.2 AA is the release target; automated checks supplement keyboard, screen-reader, switch, touch, and child testing.

## Avoid

Generic child dashboards, excessive currencies and badges, gambling-like reveals, public ranking, red-only failure, copied reference layouts, combat as the default metaphor, and Caribbean identity reduced to beaches, flags, Carnival costumes, or constant steelpan.

The current palette is an implementation trial, not a final identity. Local reference boards, contrast validation, cultural review, and child preference testing must approve it.
